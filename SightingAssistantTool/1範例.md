## 1) 使用者操作情境（CLI）

### 1.1 安裝與註冊 toolkit

```bash
dt extensions enable gnai
git clone https://github.com/intel-sandbox/SightingAssistantTool.git sighting
cd sighting
dt gnai toolkits register .
```

### 1.2 驗證 toolkit

```bash
dt gnai toolkits validate <sighting_toolkit_path>
```

### 1.3 一般 HSD 分析

```bash
dt gnai ask "Please assist with the HSD id 14025723680"
```

### 1.4 指定 assistant 的摘要情境

```bash
dt gnai ask "What is the summary of HSD id 15018275324" --assistant sighting_assistant
```

### 1.5 除錯模式（verbose）

```bash
dt gnai ask "Please assist with the HSD id 14025723680" -v
```

### 1.6 accuracy test 的批次測試情境（Notebook 內）

```bash
dt gnai ask 'Give summary of HSD id {hsd_id}' -v --assistant sighting_assistant
```

## 5) ETL 分析器的內部判定情境（自動分類）

這些是 `src/etl_classifier.py` 中定義的 Scenario 判定流程:

- Scenario 1: 偵測到 `DxgkDdiStartDevice` -> `Boottrace`
- Scenario 2: 無 boot pattern，且同時有 Intel + Media patterns -> `WPT`
- Scenario 3: 無 boot pattern，只有 Intel patterns -> `Display ETL`
- Scenario 4: 無 boot pattern，只有 Media patterns -> `GPUView`
- No matching scenario -> `Unknown`

## 7) 新手版操作手冊（何時用 / 輸入 / 預期輸出）

### 7.1 第一次安裝與註冊

何時用:

- 新機器第一次使用 SightingAssistantTool

輸入:

```bash
dt extensions enable gnai
git clone https://github.com/intel-sandbox/SightingAssistantTool.git sighting
cd sighting
dt gnai toolkits register .
```

預期輸出:

- `dt gnai toolkits register .` 成功完成
- toolkit 可在 GNAI 中被辨識

### 7.2 確認 toolkit 可用

何時用:

- 註冊後想確認設定正確

輸入:

```bash
dt gnai toolkits validate <sighting_toolkit_path>
```

預期輸出:

- 顯示 toolkit 有效（valid）
- 若失敗通常會指出 YAML 或依賴問題

### 7.3 快速做一筆 HSD 分析

何時用:

- 想先確認整體流程能跑通

輸入:

```bash
dt gnai ask "Please assist with the HSD id <HSD_ID>"
```

預期輸出:

- 看到 7 大段結構化分析（摘要、分類、附件檢查、建議等）

### 7.4 要求摘要版結果

何時用:

- 只想快速知道單號重點

輸入:

```bash
dt gnai ask "What is the summary of HSD id <HSD_ID>" --assistant sighting_assistant
```

預期輸出:

- 較聚焦在 summary / 主問題 /建議，不一定展開全部細節

### 7.5 除錯模式看完整執行細節

何時用:

- 你想知道卡在哪個 tool

輸入:

```bash
dt gnai ask "Please assist with the HSD id <HSD_ID>" -v
```

預期輸出:

- 額外顯示 tool 觸發順序、錯誤訊息、模型執行資訊

### 7.6 Game 類問題（例如效能下降）

何時用:

- HSD 描述是 FPS 降低、卡頓、遊戲崩潰、畫面破圖

輸入:

```text
Please assist with the HSD id <HSD_ID>
```

預期輸出:

- 分類為 `Gaming/Performance/Corruption` 之一
- 會回覆對應 scenario 的 isolation 步驟與 log collection BKM

### 7.7 Display 類問題（黑屏/閃爍/亮度/GOP）

何時用:

- HSD 與顯示流程、螢幕訊號、開機前 GOP 有關

輸入:

```text
Analyze HSD <HSD_ID>
```

預期輸出:

- 分類為 `Display` 路線
- 若有可用 display log，會觸發 DisplayDebugger 分析

### 7.8 BSOD/TDR/Hang 類問題

何時用:

- HSD 與 TDR、藍畫面、hang、app crash 有關

輸入:

```text
Analyze HSD <HSD_ID>
```

預期輸出:

- 分類為 `BSOD/TDR/Hard Hang/App Crash` 路線
- 提供 dump/log 收集建議；若找到 GDHM ID 會嘗試觸發 sherlog

### 7.9 Media/CP/Netflix 類問題

何時用:

- 影片卡頓、畫面破圖、CP/HWDRM/Netflix error

輸入:

```text
Please assist with the HSD id <HSD_ID>
```

預期輸出:

- 分類為 `Media` 或 `Content Protection`
- 提供對應 scenario 的 isolation 與 GPUView/WPT/GSC 等 log 建議

### 7.10 AI/ML/Content Creator 類問題

何時用:

- OpenCL/OpenGL/Vulkan/DirectML/OpenVINO 相關 workload

輸入:

```text
Analyze HSD <HSD_ID>
```

預期輸出:

- 分類為 `AI` 或 Content Creator 相關路線
- 提供對應 scenario（如 OpenCL、Vulkan、DirectML）的 BKM 建議

### 7.11 Yellow Bang/WHQL/Installer/IGS/SnS/GuC 類問題

何時用:

- 驅動安裝失敗、Yellow Bang、WHQL/Compliance、長時間壓測異常

輸入:

```text
Please assist with the HSD id <HSD_ID>
```

預期輸出:

- 分類為對應路線（Yellow Bang/WHQL/Installer/IGS 等）
- 回傳對應 scenario 的隔離步驟與 log 收集清單
