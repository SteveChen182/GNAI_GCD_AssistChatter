## 使用限制與成本追蹤

### 說明
ExpertGPT 現在會依「不同存取方式」分別追蹤每日使用成本，確保公平使用與系統效能。每種存取方式都有各自的每日上限，並於午夜重置。

### 重要
這些限制是依 API 呼叫成本計算，不是請求次數。不同 AI 模型單次成本不同，因此你每天可發送的請求數會有差異。

### 網頁平台使用（Web Platform Usage）
你直接在 ExpertGPT 網站主聊天介面或 Expert 建立頁聊天時。

- **每日上限**：$5.00 / 天
- **每天午夜自動重置**
- **可查看位置**：
	- 主聊天頁（`index.html`）
	- Expert 建立/測試頁（`create.html`）
	- 任何直接網站互動頁

**小提醒**：你可在聊天頁右上角查看目前平台使用量；進度條會顯示你已用掉多少每日 $5.00。

### 個人 API Key 使用（Personal API Keys Usage）
使用你的個人 API key，透過 OpenAI 相容 API 進行程式化存取時。

- **每日上限**：預設 $5.00 / 天
- 可申請提高
- 每天午夜重置
- **常見使用情境**：
	- Python 腳本與應用程式
	- 第三方工具（LangChain、Pandas AI 等）
	- 客製整合與自動化
	- 開發與測試

### 如何申請提高限制
1. 到 `Request Quota Increase` 頁面。
2. 填寫申請表，包含：
	- 你希望的每日上限（例如 $10、$15）
	- 商業用途說明
	- 預計期間（暫時或永久）
	- 緊急程度
3. 送出等待管理員審核。
4. 在同頁追蹤申請狀態。
5. 核准後新上限立即生效。

**注意**：通常 1-2 個工作天審核完成。較高上限可能需要額外核准以利預算規劃。

### API Experts Route 使用
使用舊版 `/api/experts` 端點進行程式化存取（多為既有整合）。

- **每日上限**：$5.00 / 天
- 屬於舊端點，建議遷移到 Personal API Keys

**特殊豁免**：部分服務帳號可無上限使用此端點（關鍵業務用途）。若你需要無上限，請向 ExpertGPT 團隊提出商業需求說明。

### 遷移建議
- 改用 Personal API Keys（可取得更好的限制彈性）
- OpenAI 相容格式
- 更好的錯誤處理與功能
- 支援串流與較新的能力

### 使用成本如何計算
成本取決於你使用的模型類別與 token 量（可粗略視為字數）。

| 模型類別 | 每 1K Tokens 成本 | 範例 |
|---|---:|---|
| 輕量模型 | $0.005 - $0.008 | $1 約可用 200,000 tokens |
| 標準模型 | $0.010 - $0.012 | $1 約可用 100,000 tokens |
| 進階模型 | $0.015 - $0.030 | $1 約可用 50,000 tokens |

**快速估算**：約 1,000 tokens ≈ 750 英文字。若使用 `gpt-4.1-mini`（$0.010 / 1K tokens），一段 750 字的對話大約是 1 美分。

### 如何管理你的使用量
- **監控限制**
	- **Web 平台**：看聊天頁右上角指示器
	- **API 使用**：從 Personal API Keys 頁下載並執行 quota checker script
- **通知**
	- 接近上限時會收到警示
- **最佳化建議**
	- 簡單任務用較輕模型（如 `gpt-4.1-mini`, `o3-mini`）
	- 提示詞精簡聚焦
	- 對重複問題快取回應
	- 類似請求批次處理

### 達到上限時會怎樣
- 新請求會暫時被阻擋
- 午夜會自動重置
- 可申請提高上限
- 不會影響其他使用者與系統效能

### 準備開始？
善用 ExpertGPT 彈性的用量限制系統，掌握你的 AI 使用成本。

- 取得 API Key
- 申請提高限制
- 查看個人檔案
